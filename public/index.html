Built on Celo

Native Celo infrastructure.
Not bolted on.

Every layer of Pulse uses existing Celo ecosystem tools — not custom-built replacements.

ERC-8004 Registry

Pulse is a registered ERC-8004 agent (ID #9358). Every alert writes a feedback receipt to the Reputation Registry — building verifiable onchain history.

Mento SortedOracles

Live CELO/USD price fetched directly from Mento's onchain oracle. No third-party price API, no stale data — the same source Mento itself uses.

Self Protocol

ZK passport verification via Self Protocol. Users prove humanity without revealing identity. Pulse verifies proofs against Celo's IdentityVerificationHub contract.

Native token monitoring

Watches CELO, cUSD, cEUR, and bridged USDC — the core Celo token ecosystem — with no dependency on external price feeds or indexers.

Forno RPC

Connected to Celo's own Forno RPC endpoint. Scans every block for whale activity within seconds of it being finalized on the L2.

eth-crypto encryption

User emails are encrypted client-side with the agent's public key before storage. Decrypted in-memory at send time, then wiped. The same privacy architecture Herald uses on Solana.

Under the hood

One agent.
Every alert onchain.

Every time Pulse sends an alert, it calls giveFeedback() on the ERC-8004 Reputation Registry — writing an immutable receipt of the agent's activity onchain.

This means every alert Pulse sends is verifiable, auditable, and contributes to the agent's onchain reputation score on 8004scan.

View receipts on 8004scan ↗
onchain.ts 
// Write receipt per alert → ERC-8004
const contract = new Contract(
  REPUTATION_REGISTRY, ABI, wallet
);

const tag = ethers.encodeBytes32String(
  alertType
);

const tx = await contract.giveFeedback(
  9358,      // Pulse agent ID
  [tag],     // alert type tag
  [1],       // positive signal
  comment    // human-readable note
);

await tx.wait();
// Receipt written. Permanent.
